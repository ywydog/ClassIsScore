# 修仙主题包 Spec

## Why
将 ClassIsScore 教室积分管理系统升级为修仙风格，提供全局可切换的修仙主题包，包含术语映射、古风视觉、道友境界体系、仙宠渡劫机制等，增强趣味性和沉浸感。

## What Changes
- 新增修仙主题包（独立于默认主题），包含术语映射、古风样式、组件覆盖
- 新增道友境界体系（64级，直接搬诸天修仙练气境界，原始阈值）
- 新增仙宠渡劫机制（5级/8级渡劫，动态成功率，阶梯式失败惩罚）
- 新增道友突破机制（修为达到阈值时触发突破提示）
- 修仙模式下仙界展示替代原大屏展示
- 修为计算：学生积分 × (1 + 仙宠加成%) + 仙宠等级 × 固定值
- 设置中新增主题切换开关

## Impact
- Affected code:
  - `client/src/types/index.ts` — 新增 ThemeMode、CultivationLevel 等类型
  - `client/src/stores/settings.ts` — 主题切换逻辑
  - `client/src/views/display/ScoreDisplay.vue` — 仙界展示覆盖
  - `client/src/views/admin/ScoreManagement.vue` — 修炼管理术语
  - `client/src/views/admin/Settings.vue` — 主题切换入口
  - `client/src/components/display/PetDisplay.vue` — 仙宠展示覆盖
  - `client/src/components/score/ExportReportDialog.vue` — 仙册录录术语
  - `client/src/utils/petSystem.ts` — 仙宠等级称号、渡劫逻辑
  - `client/src/views/admin/StudentProfile.vue` — 道友档案术语
  - 新增 `client/src/themes/xianxia/` 目录 — 修仙主题包

## ADDED Requirements

### Requirement: 修仙主题包架构

系统 SHALL 提供独立主题包机制，修仙主题包位于 `client/src/themes/xianxia/`，包含：
- `terminology.ts` — 术语映射表
- `styles.css` — 古风样式覆盖
- `components/` — 修仙风格组件覆盖
- `assets/` — 古风资源（背景、边框、特效素材）

#### Scenario: 主题切换
- **WHEN** 用户在设置中切换主题为"修仙模式"
- **THEN** 所有页面术语、视觉风格、组件渲染自动切换为修仙风格
- **AND** 数据层完全共享，只有展示层不同

#### Scenario: 主题恢复
- **WHEN** 用户在设置中切换回"默认模式"
- **THEN** 所有页面恢复为原始术语和视觉风格

### Requirement: 修为计算

修仙模式下，道友的修为 SHALL 按以下公式计算：

```
修为 = 学生学期总积分 × (1 + 仙宠加成%) + 仙宠等级 × 5
```

仙宠加成%按等级递增：
- 1级(仙胎): +3%
- 2级(仙仔): +6%
- 3级(仙兽): +9%
- 4级(仙道): +12%
- 5级(仙灵): +15%
- 6级(仙王): +18%
- 7级(仙帝): +21%
- 8级(仙尊): +24%

#### Scenario: 修为显示
- **WHEN** 修仙模式启用
- **THEN** 仙界展示页面中每位道友显示：境界名称 + 修为数值
- **AND** 修为数值包含仙宠加成

### Requirement: 仙宠渡劫系统

仙宠升级到5级(仙灵)和8级(仙尊)时，SHALL 触发渡劫机制：

**成功率计算**：
```
基础成功率 = 50%（5级渡劫）/ 30%（8级渡劫）
经验加成 = (当前经验 / 升级所需经验) × 30%
最终成功率 = min(基础成功率 + 经验加成, 80%)
```

**失败惩罚**（参考诸天修仙阶梯式惩罚）：

| 随机值范围 | 惩罚 | 描述文案 |
|-----------|------|---------|
| > 0.9 | 扣当前经验40% | 天劫降世，道基崩碎！ |
| 0.7-0.9 | 扣当前经验20% | 雷劫反噬，修为受损 |
| 0.5-0.7 | 扣当前经验10% | 心魔侵扰，略有损耗 |
| 0.3-0.5 | 扣当前经验5% | 劫雷擦身，轻微反噬 |
| < 0.3 | 不扣经验 | 天劫未成，全身而退 |

#### Scenario: 渡劫触发
- **WHEN** 仙宠经验达到升级所需经验，且目标等级为5级或8级
- **THEN** 升级按钮变为"渡劫"按钮
- **AND** 点击渡劫弹出确认面板，显示：当前成功率、可能惩罚、渡劫等级

#### Scenario: 渡劫成功
- **WHEN** 用户确认渡劫且随机数 <= 成功率
- **THEN** 播放渡劫成功动画（雷劫特效+金光）
- **AND** 仙宠升级到新等级，获得新称号

#### Scenario: 渡劫失败
- **WHEN** 用户确认渡劫且随机数 > 成功率
- **THEN** 播放渡劫失败动画
- **AND** 根据随机值扣除对应比例的经验
- **AND** 显示失败描述文案
- **AND** 可重新积累经验再次尝试渡劫

### Requirement: 道友突破机制

道友的修为达到新境界阈值时，SHALL 触发突破提示：

**突破判定**：
- 修为达到新境界阈值时自动触发
- 突破为确定性事件（不需要概率判定，与仙宠渡劫不同）
- 弹出突破提示弹窗，显示：
  - 旧境界 → 新境界
  - 修为变化
  - 突破动画效果

**特殊境界突破**：
- 无妄真劫境(level_id=41)：需要渡劫确认（类似仙宠渡劫）
  - 成功率 = 50% + (修为超出阈值部分 / 下一级阈值) × 30%，上限80%
  - 失败惩罚：扣修为的20%
- 地仙及以上(level_id>=42)：每次突破都需要渡劫确认
  - 成功率 = max(30%, 80% - level_id × 1%)
  - 失败惩罚：扣修为的10%-40%（阶梯式）

#### Scenario: 普通突破
- **WHEN** 道友修为达到新境界阈值且新境界level_id < 41
- **THEN** 弹出突破提示，显示突破动画
- **AND** 突破为确定性成功

#### Scenario: 渡劫突破
- **WHEN** 道友修为达到新境界阈值且新境界level_id >= 41
- **THEN** 弹出渡劫确认面板，显示成功率
- **AND** 需要用户确认后进行概率判定
- **AND** 成功则突破，失败则扣除修为

### Requirement: 仙界展示页面

修仙模式下，仙界展示 SHALL 替代原大屏展示页面：

- 学生积分页：强制使用卡片展示模式
  - 每张卡片显示：道友名称、境界名称、修为数值、仙宠信息
  - 积分展示为整个学期总积分
- 小组积分页：术语替换（宗门、弟子、掌门）
- 宠物页：术语替换为仙宠，等级称号替换

#### Scenario: 仙界展示
- **WHEN** 修仙模式启用且用户进入大屏展示
- **THEN** 页面标题显示"仙界展示"
- **AND** 学生积分页使用卡片模式，显示境界+修为
- **AND** 小组页显示"宗门"相关术语
- **AND** 宠物页显示"仙宠"相关术语

### Requirement: 古风视觉风格

修仙模式下 SHALL 应用全面古风视觉改造：

**配色方案**：
- 主色：#8B5CF6（紫色）→ #C9A84C（金色）
- 辅色：#6366F1（靛蓝）→ #7C3AED（深紫）
- 背景：白色 → 深色古风（#1A1A2E 深蓝黑 + 半透明云纹）
- 文字：深灰 → 金色/白色
- 边框：灰色 → 金色古风纹样

**特效**：
- 仙宠展示：仙气缭绕动画（CSS粒子效果）
- 排行榜/仙榜：金色光晕 + 云纹装饰
- 悟道（加分）动画：灵力上升光效
- 魔障（减分）动画：暗色特效
- 背景：缓慢飘动的云雾

**仙宠等级边框颜色**：
- 仙胎(1级)：灰白
- 仙仔(2级)：浅蓝
- 仙兽(3级)：蓝色
- 仙道(4级)：青色
- 仙灵(5级)：紫色
- 仙王(6级)：粉红
- 仙帝(7级)：红色
- 仙尊(8级)：金色

### Requirement: 设置中主题切换

系统 SHALL 在设置页面提供主题切换选项：

- 位置：通用设置（道法设置）区域
- 选项：默认模式 / 修仙模式
- 切换后立即生效，无需刷新页面
- 主题偏好保存到 AppSettings

#### Scenario: 切换到修仙模式
- **WHEN** 用户在设置中选择"修仙模式"
- **THEN** 所有页面立即切换为修仙风格
- **AND** 设置保存到后端

#### Scenario: 切换回默认模式
- **WHEN** 用户在设置中选择"默认模式"
- **THEN** 所有页面立即恢复为原始风格
